## 1) Ce qui compte vraiment pour choisir un plan
Le piège classique : choisir “au doigt mouillé”, puis se retrouver bloqué par :
- le volume mensuel (mots / messages / crédits),
- les fonctionnalités débloquées uniquement sur certains niveaux (ex. KoalaLinks / fonctions avancées),
- l’écart de consommation selon les modèles utilisés.

La bonne approche : choisir un plan sur **votre rythme de publication** et **votre process** (relecture, fact-check, maillage, publication).

## 2) Logique de choix (décision en 60 secondes)
### Étape A — Votre volume d’articles / mois
- 1–4 articles/mois : “usage léger”
- 5–20 articles/mois : “usage régulier”
- 20+ articles/mois : “production en volume”

### Étape B — Votre besoin de maillage interne
- Site jeune / peu de pages : maillage automatique peu prioritaire
- Site déjà fourni : KoalaLinks peut devenir un accélérateur

➡️ Guide : [koala-koalalinks-internal-linking.md](./koala-koalalinks-internal-linking.md)

### Étape C — Votre besoin d’automatisation / publication
- simple copier-coller : OK
- publication 1-clic / webhooks / API : utile si production régulière

➡️ Guide : [koala-integrations-wordpress-shopify-webflow.md](./koala-integrations-wordpress-shopify-webflow.md)

## 3) Scénarios concrets (selon volume)
### Scénario 1 — “Petit site / blog de niche” (cadence faible)
Objectif : produire des contenus piliers + quelques articles support.  
Recommandation : privilégier la qualité (plan + relecture) plutôt que la vitesse brute.

**À faire :**
- 1 article pilier / semaine ou / 2 semaines
- 1 FAQ solide
- maillage interne manuel (au début)

### Scénario 2 — “Éditeur SEO” (cadence régulière)
Objectif : produire à la chaîne, sans perdre la cohérence.  
Ici, l’intérêt de Koala devient plus net :
- production standardisée,
- intégrations,
- (selon plan) KoalaLinks pour le maillage.

**À faire :**
- templates (types d’articles)
- checklists “fact-check” répétables
- liens internes systématiques

### Scénario 3 — “Affiliation / comparatifs”
Objectif : créer des pages money + pages support (guides).  
Le point sensible : rester crédible (pas de promesses, données vérifiées, sections utiles).

**À faire :**
- pages “comment choisir”
- pages “alternatives”
- pages “erreurs à éviter”
- tableaux / critères de choix

## 4) Mensuel vs annuel (quand l’annuel a du sens)
L’annuel peut être intéressant si :
- production stable sur plusieurs mois,
- besoin de crédits utilisables sur la durée,
- volonté de lisser les coûts.

Le mensuel est souvent préférable si :
- test / phase d’exploration,
- cadence irrégulière,
- pivot de stratégie possible.

## 5) FAQ prix
### Est-ce qu’un petit plan suffit ?
Oui si la cadence est faible et que le process est propre (plan + relecture). Sinon, la frustration vient vite du volume.

### KoalaLinks est indispensable ?
Non. Utile surtout quand le site a déjà un stock de pages. Sur un site jeune, le gain est mécaniquement limité.

### Quel est le meilleur “ROI” ?
Le ROI dépend surtout du process :
- si publication sans relecture → ROI faible (contenu moyen)
- si process standardisé → ROI beaucoup plus élevé
